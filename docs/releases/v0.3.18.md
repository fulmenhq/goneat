# Goneat v0.3.18 — Checkmake Config Overrides (MVP)

**Release Date**: 2025-12-13
**Status**: Draft

## Overview

Goneat v0.3.18 adds minimal but high-impact configuration support for Makefile linting by generating a temporary `checkmake` config file from `.goneat/assess.yaml` and invoking `checkmake --config`.

This enables teams to reduce noise from checkmake defaults while keeping Makefile linting as a first-class quality gate.

## ✅ Supported checkmake overrides

Configure in `.goneat/assess.yaml`:

```yaml
version: 1
lint:
  make:
    checkmake:
      enabled: true
      config:
        max_body_length: 15
        min_phony_targets: ["all", "clean", "test"]
    paths: ["**/Makefile"]
    ignore: []
```

- `max_body_length` maps to checkmake rule `maxbodylength` (`maxBodyLength=<n>`)
- `min_phony_targets` maps to checkmake rule `minphony` (`required=a,b,c`)

## Notes

- Goneat still controls file scope using `.gitignore`/`.goneatignore` (v0.3.17+).
- Other checkmake rules may remain tool-defined and not configurable.
